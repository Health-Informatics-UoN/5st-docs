## Download the deployment files

There are a few ways to download the deployment files.

<Tabs items={["HTTPS Clone", "SSH Clone", "GitHub CLI", "Download Zip"]}>
  <Tabs.Tab title="HTTPS Clone">
    Clone using the web URL. 
    
    ```bash copy
    git clone https://github.com/SwanseaUniversityMedical/5S-TES-deployment.git
    ```
    
  </Tabs.Tab>
  <Tabs.Tab title="SSH Clone">
    Clone using a password-protected SSH key.
    
    ```bash copy
    git clone git@github.com:SwanseaUniversityMedical/5S-TES-Deployment.git
    ```
    
  </Tabs.Tab>
  <Tabs.Tab title="GitHub CLI">
    Clone using the GitHub CLI.
    
    ```bash copy
    gh repo clone SwanseaUniversityMedical/5S-TES-Deployment
    ```
    
  </Tabs.Tab>
  <Tabs.Tab title="Download Zip">
   - Download the repository as a zip file.
    - Go to the [repository page](https://github.com/SwanseaUniversityMedical/5S-TES-Deployment) and click on the "Code" button.
    - Then click on the "Download ZIP" button.
    - Save the file to your local machine.
    - Unzip the file.
    - The repository will be in the `5S-TES-Deployment` directory.
   
   - Alternatively, run the command:
    ```bash copy
    curl -L -o 5S-TES-Deployment.zip https://github.com/SwanseaUniversityMedical/5S-TES-Deployment/archive/refs/heads/main.zip
    ```
    
  </Tabs.Tab>
</Tabs>

### Prepare the necessary files

If you would like to modify the services while keeping the original files structure (to make Git tracking easier, for example), you can follow the steps below.
- Create a new `compose.yml` file next to `DeploymentStack` and `ServiceStack` directories.

<FileTree>
    <FileTree.File name="compose.yml" />
    <FileTree.Folder name="DeploymentStack" />
    <FileTree.Folder name="ServiceStack" />
</FileTree>

- Add the following content to the `compose.yml` file, to make the connection with the original TRE's docker-compose.yml file:

```yaml copy
include:
    - DeploymentStack/TRE/docker-compose.yml
```

- Add the override settings to the `compose.yml` file if there are any. For example, if you want to change the port of the TRE Agent UI to `8123`, the content of `compose.yml` should be:

```yaml copy
include:
    - DeploymentStack/TRE/docker-compose.yml

services: 
  tre-ui:
    ports: !override # To override the port of the TRE Agent UI, to 8123 in this example
      - 8123:8080
```


If you would like to change the directory structure (e.g., to have your own way of managing the files), there are some notes below.
- We recommend to move the following files and directories to a separate directory, e.g., `/srv/tre`:

    - `DeploymentStack/TRE/docker-compose.yml`
    - `DeploymentStack/TRE/config/`
    - `ServiceStack/`

The `docker-compose.yml` file in the `DeploymentStack/TRE/` directory includes the services for the TRE Agent and Data Egress using relative paths to the compose manifests in the `ServiceStack/` directory, where they are defined.
The services inside the `ServiceStack/` directory are using the configuration files in the `config/` directory via `CONFIG_PATH` environment variable.

<Callout type="info">
`CONFIG_PATH` is the relative path from the services (yaml files) inside the `ServiceStack/` directory to the configuration files (which is under `config/` directory).
For example, the `CONFIG_PATH` for the scenario below is `../../../config` because `platform.yml` is nested 3 levels deep from the `ServiceStack/` directory compared to the `config/` directory.

<FileTree>
    <FileTree.File name="docker-compose.yml" />
    <FileTree.Folder name="ServiceStack" >
        <FileTree.Folder name="compose-manifests" >
            <FileTree.Folder name="application" >
                <FileTree.File name="platform.yml" />
            </FileTree.Folder>
        </FileTree.Folder>
    </FileTree.Folder>
    <FileTree.Folder name="config">
        <FileTree.File name="config.json" />
    </FileTree.Folder>
</FileTree>

</Callout>

- Since the directory structure is changed, you need to update the relative paths in the `docker-compose.yml` and `CONFIG_PATH` in the `.env` file accordingly.
